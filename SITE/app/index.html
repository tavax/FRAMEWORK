Les concepts de base de l'utilisation du framework

Où récupérer les sources ?

Un dépot GIT Cabestan a été mis en place et tout y est accessible d'un simple download.

Comment se connecter ?

Il faut se rendre à l'adresse du dépot disponible ici et cliquer sur le bouton pour y récupérer la ligne suivante :

https://github.com/tavax/FRAMEWORK.git

Tooltip on top

Comment utiliser cette URL ?

l'extension .git fonctionne en ligne de commande. Pour que cela soit utilisable, il faut installer sur le poste de travail plusieurs choses :

  • Télécharger et installer node.js (nécessite les droits admin de la machine pour son installation).
  • Télécharger et installer Git.

Mise en place de l'environnement de travail

Une fois les sources en place et node.js et git installer, il faut récupérer les dépendances des scripts. un clic droit dans le dossier et sélectionner la ligne "Git Bash Here".

dans la fenêtre qui va s'ouvrir, la ligne de commande suivante va s'en charger.

npm install

Procédure de travail (nouveau dépot)

Le dossier est à jour.

La première chose à faire et le plus important : la création de branche. Rien de plus simple, la commande suivant vas vous montrer toutes les branches que vous avez réalisée

git branch

On ne travaille JAMAIS sur la branche master d'un dépot, cette branche est votre backup de secours, elle doit toujours être à jour par rapport au dépot distant.

Procédure de travail (mettre à jour la branche master)

La ligne de commande suivante vous permets de faire un état des lieux du code, à la fois en local et distant du dépot

git status

La ligne de commande suivante va récupérer le code distant et mettre à jour votre branche (ici master).

git pull origin master

Plusieurs états s'offrent à vous :

  • nothing to commit, working tree clean : Votre branche est à jour, parfait.
  • Changes not staged for commit: : Vous avez fait des modifications. il faudra les envoyer sur le dépot distant une fois celles-ci terminées.

Procédure de travail (création d'une branche à partir du master)

Afin de préserver son code, il faut toujours créer des branches à partir du master afin d'être sur que tout soit à jour avant de faire quelque chose. Faire la commande suivante depuis la branche master

git checkout -b < nom-de-la-branche >

Cette manip est en réalité un mixe de plusieurs autres, est à l'avantage de faire gagner du temps et d'être immédiatement transférer sur la branche nouvellement créée.

Le nom de branche doit être relatif au travail que vous allez faire, afin de facilité le traitement collaboratif mais aussi pour se retrouver dans votre travail en local. par ex. après un git status.

pour sortir de votre branche, un simple et le tour est joué

git checkout < nom-de-la-branche-locale-desiree >

Procédure de travail (envoyer votre code sur le dépot distant)

Vos modifications sont terminées, il est temps d'envoyer votre code pour que le dépot distant reflète votre travail.

step 1

git add . Cela vas mettre en "cache" toutes les modifications grâce au . , sinon, on peut mettre fichier par fichier, dans ce cas, la commande suivante git add < nom-du-fichier-modifié >

step 2

git commit -m "informations utiles en rapport au travail fait"

Ex. git commit -m "mise à jour du bloc X qui corrige le bug sur gmail app"

step 3

git push orign < nom-de-votre-branche >

Vous validez vos modifications sur le dépot distant, tout en créant une branche sur celui-ci.

Procédure de travail (récupérer une branche distante)

avant toute chose, placez vous sur votre branche master et procédez ainsi

step 1

git checkout -b < nom-branche-local-qui-sera-créer > origin/< nom-branche-distance-désirée >

Cela va créer une nouvelle branche locale intitulée nom-branche-local-qui-sera-créer qui est une copie d’une branche distante du dépôt origin. Et vous placez directement sur cette branche.
